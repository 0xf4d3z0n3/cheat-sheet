🔍 Useful Forensic SQL Queries

Chromium — Full browsing history with visit details


SELECT
    urls.url,
    urls.title,
    urls.visit_count,
    urls.typed_count,
    datetime(visits.visit_time / 1000000 - 11644473600, 'unixepoch', 'localtime') AS visit_date,
    visits.visit_duration / 1000000.0 AS duration_seconds,
    visits.transition & 0xFF AS core_transition,
    CASE visits.transition & 0xFF
        WHEN 0 THEN 'LINK'
        WHEN 1 THEN 'TYPED'
        WHEN 2 THEN 'AUTO_BOOKMARK'
        WHEN 3 THEN 'AUTO_SUBFRAME'
        WHEN 4 THEN 'MANUAL_SUBFRAME'
        WHEN 5 THEN 'GENERATED'
        WHEN 6 THEN 'START_PAGE'
        WHEN 7 THEN 'FORM_SUBMIT'
        WHEN 8 THEN 'RELOAD'
        WHEN 9 THEN 'KEYWORD'
        WHEN 10 THEN 'KEYWORD_GENERATED'
        ELSE 'OTHER'
    END AS transition_type,
    visits.from_visit
FROM urls
JOIN visits ON urls.id = visits.url
ORDER BY visits.visit_time DESC;

Chromium — Downloads with redirect chains


SELECT
    d.id,
    d.tab_url,
    d.referrer,
    duc.url AS download_url,
    duc.chain_index,
    d.target_path,
    d.total_bytes,
    d.mime_type,
    datetime(d.start_time / 1000000 - 11644473600, 'unixepoch', 'localtime') AS start_time,
    datetime(d.end_time / 1000000 - 11644473600, 'unixepoch', 'localtime')   AS end_time,
    d.danger_type,
    d.opened
FROM downloads d
JOIN downloads_url_chains duc ON d.id = duc.id
ORDER BY d.start_time DESC, duc.chain_index ASC;

Firefox — Bookmarks with folder tree


WITH RECURSIVE tree(id, title, fk, parent, depth, path) AS (
    SELECT id, title, fk, parent, 0, title
    FROM moz_bookmarks WHERE parent = 1
    UNION ALL
    SELECT b.id, b.title, b.fk, b.parent, t.depth + 1, t.path || '/' || b.title
    FROM moz_bookmarks b JOIN tree t ON b.parent = t.id
)
SELECT
    tree.path,
    p.url,
    datetime(tree.fk, 'unixepoch', 'localtime') AS date_added
FROM tree
LEFT JOIN moz_places p ON tree.fk = p.id
WHERE tree.fk IS NOT NULL
ORDER BY tree.path;

⏱️ Timestamp Conversions

Browser / Format Epoch Resolution Conversion to Unix Epoch
Chromium (WebKit/Chrome) 1601-01-01 00:00:00 UTC Microseconds unix_ts = (chrome_ts / 1,000,000) - 11,644,473,600
Firefox 1970-01-01 00:00:00 UTC Microseconds (most fields) unix_ts = ff_ts / 1,000,000
Firefox cookies.expiry 1970-01-01 00:00:00 UTC Seconds Already Unix
Safari (Core Data / Cocoa) 2001-01-01 00:00:00 UTC Seconds (float) unix_ts = safari_ts + 978,307,200
Internet Explorer / Edge (FILETIME) 1601-01-01 00:00:00 UTC 100-nanosecond intervals unix_ts = (filetime / 10,000,000) - 11,644,473,600
Internet Explorer (OLE / FAT) Various Varies Use oletools or manual conversion

Quick Python Helpers


from datetime import datetime, timedelta, timezone

# Chromium timestamp → datetime
def chrome_time(ts):
    epoch = datetime(1601, 1, 1, tzinfo=timezone.utc)
    return epoch + timedelta(microseconds=ts)

# Firefox timestamp (microseconds) → datetime
def firefox_time(ts):
    return datetime.fromtimestamp(ts / 1_000_000, tz=timezone.utc)

# Safari / Cocoa timestamp → datetime
def safari_time(ts):
    epoch = datetime(2001, 1, 1, tzinfo=timezone.utc)
    return epoch + timedelta(seconds=ts)

# Windows FILETIME → datetime
def filetime_to_dt(ft):
    epoch = datetime(1601, 1, 1, tzinfo=timezone.utc)
    return epoch + timedelta(microseconds=ft // 10)

🔐 Encryption & Protected Data

Chromium Password / Cookie Decryption

OS Method Key Location
Windows AES-256-GCM; key in Local State (JSON → os_crypt.encrypted_key), itself protected by DPAPI (CryptUnprotectData) User Data\Local State
macOS AES-128-CBC; key stored in macOS Keychain under Chrome Safe Storage (or Chromium Safe Storage, Brave Safe Storage, etc.) Keychain

Tool: AES-GCM Chrome decrypt, mimikatz dpapi::chrome, or hindsight.

Firefox Password Decryption

  • Master password → PKCS#5 PBKDF2 key → Decrypts key4.db (NSS pkcs11.txt).
  • If no master password is set, decryption is trivial with tools.
  • Tools: firefox_decrypt (Python), nss library directly, LaZagne.

Safari Password Decryption

  • Passwords stored in macOS Keychain (login.keychain-db).
  • Extraction requires user login password or Keychain unlock.
  • Tools: security find-internet-password -g, chainbreaker, keychain-dumper (jailbroken iOS).

Cookies.binarycookies (Safari)

  • Custom binary format (not encrypted, but binary-packed).
  • Tools: BinaryCookieReader.py, mac_apt, APOLLO.
